# AI 短剧剧集前端 UI/UX 全流程设计描述

> 状态：待 Review
> 来源参考：`/Users/leo/cowork/short-drama-learn/short-drama-frontend-spec.md`
> 说明：本文档仅描述前端 UI/UX 与实现结构，不代表立即实施计划。Review 修改完成后，可作为后续实现依据。

## 1. 总体定位

将系统设计为“剧集生产工作台”：用户从项目首页进入某个剧集项目后，围绕单集或多集完成剧本、分镜、资产、图片融合、视频、音频、剪辑、导出全流程。

整体体验以“阶段进度 + 逐镜头审核 + AI 批量生成队列”为核心，既能一键推进，也能让创作者在关键节点精修。

默认定位为桌面端生产工具，主界面采用顶部项目栏、左侧阶段导航、中央工作区、右侧检查/生成/版本面板的高密度布局。移动端只保留预览、审核、状态查看和轻量编辑能力。

## 2. 核心 UI 结构

- 首页展示项目卡片、新建项目入口、最近更新、当前阶段、集数、平台和画幅。
- 新建项目时锁定风格词、目标平台、集数、单集时长和画幅。
- 项目工作台顶部常驻阶段进度条，显示每阶段状态：未开始、进行中、待审核、已锁定、异常。
- 点击阶段可跳转，前置未完成时显示风险确认。
- 增加“集数选择器”：在工作台顶部或左侧显示第 1 集至第 N 集，每集有独立进度。
- 角色、场景、风格板为项目级共享资产；剧本、分镜、视频、音频、剪辑为单集级内容。
- 每个 AI 生成动作统一使用“提示词可编辑 + 参数选择 + 生成队列 + 多版本结果 + 审核锁定”模式。
- 所有核心产物都具备状态流转：草稿、生成中、待审核、已通过、已拒绝、已锁定。
- 锁定后的产物如需修改，需要解锁并提示影响范围。

## 3. 全流程 UI/UX

### 3.1 拉片分析

左侧展示视频播放与关键帧，右侧展示拉片报告、叙事结构、粗分镜和创作方向。

用户上传参考视频后点击 AI 解析，系统生成拉片报告与关键帧。用户审核并修正报告后确认，分析结果进入剧本阶段，作为剧本生成输入。

关键体验要求：

- 视频播放区支持播放、暂停、倍速、逐帧查看。
- 关键帧可点击放大，并能关联到报告中的叙事节点。
- 拉片报告支持 Markdown 展示和编辑。
- 粗分镜表可手动增删改。
- 用户确认前，下一阶段应显示“待确认输入”状态。

### 3.2 剧本编辑器

左侧为 Game 点、受众、节奏、悬念、时长等参数；中央为结构化剧本编辑器；右侧为 3 版生成结果、叙事节点标注和版本对比。

用户填写参数后点击 AI 生成，系统生成多版剧本。用户选择、合并或编辑后，系统自动标注开场钩子、矛盾点、升级点、反转点和结尾钩子。确认后生成分镜。

关键体验要求：

- 剧本正文支持场景、角色、台词、动作、情绪等结构化标记。
- 多版本剧本支持左右对比、局部合并和标记当前使用版本。
- 叙事节点以颜色或标签形式嵌入正文。
- 生成分镜前提示用户确认剧本版本。

### 3.3 分镜编辑器

分镜编辑器提供缩略图列表、卡片视图、表格视图和时间线视图。

单镜头编辑包含景别、视角、时长、运镜、静态画面、动态效果、情绪、台词、音效建议和生成提示词。用户可进行二次拆解、逐镜头审核和质量检查。

关键体验要求：

- 缩略图列表用于快速定位镜头。
- 表格视图用于批量查看和编辑。
- 卡片视图用于逐镜头精修。
- 时间线视图用于检查节奏和镜头长度。
- 质量检查覆盖景别连续、视角丰富度、情绪波动、轴线规则、缺失字段。
- 检查结果提供一键修复、手动修改、忽略并继续。

### 3.4 资产管理

资产按角色、场景、道具、风格板分组。

角色展示四视图、表情、姿态和设定信息；场景展示远景、中景、特写和多角度；道具展示关键物件设定；风格板承载项目级视觉常量。

关键体验要求：

- 从分镜表自动提取角色、场景和道具列表。
- 角色资产支持正面、左侧、右侧、背面、表情设定、姿态设定。
- 场景资产支持远景、中景、特写和多角度选择。
- 一致性检查标记外形、透视、光线、色彩风格问题。
- 通过审核后资产锁定，后续图片融合和视频生成强制引用。

### 3.5 图片融合与视频生成

图片融合和视频生成围绕镜头队列展开。左侧为镜头队列和生成状态，中央为参考图、角色图、场景图、融合结果或视频预览，右侧为提示词、模型参数、质量检查和版本历史。

图片融合阶段，系统根据分镜自动匹配角色、场景和道具。用户确认前景、中景、后景的分层站位后生成分镜图。

视频生成阶段，用户选择首帧、首尾帧或文生视频等生成方式。系统读取分镜图和运镜信息生成视频提示词，并调用视频生成能力。

关键体验要求：

- 支持批量生成和逐镜头生成。
- 每个镜头都能单独重生、批准、跳过。
- 失败镜头在队列中明确标记失败原因和重试入口。
- 视频质量检查标记画面崩坏、尾部变形、主体漂移等问题。
- 通过审核的视频片段进入剪辑时间线。

### 3.6 音频工作台

音频工作台以台词配音、BGM、音效三个标签组织。

台词配音从分镜表提取台词，生成 3 版口语化结果并可试听选择。用户可调整音色、语速、情绪强度，并将实际音频时长回写分镜。

BGM 按场景生成，音效按镜头推荐，转场处自动添加转场音效。

关键体验要求：

- 台词、BGM、音效分别可试听、替换、重生和锁定。
- 配音参数包括音色、语速、情绪。
- 音量混合器至少包含台词、音效、BGM 三层。
- 实际音频时长需要影响剪辑时间线中的镜头节奏。

### 3.7 剪辑时间线

剪辑时间线包含视频、台词、BGM、音效、字幕和转场多轨结构。

上方为成片预览，下方为时间线，侧边为节奏分析。用户可自动排列素材，也可进行裁切、变速、字幕、转场、调色和精修自检。

关键体验要求：

- 时间线支持多轨预览和基础拖拽调整。
- 自动排列根据镜头顺序、视频时长和音频时长生成初始剪辑。
- 节奏分析覆盖黄金 3 秒、7 秒切镜、15 秒高潮、景别衔接、情绪波动、总时长。
- 用户可从节奏分析问题直接定位到对应镜头。
- 导出前必须完成精修自检。

### 3.8 发布与导出

发布与导出页先展示自检清单，再展示导出选项。

自检包括角色一致性、音画同步、BGM 连续性、字幕错别字、时长、画幅、音量层级。通过后用户选择 MP4、剪映项目、PR 项目或 DaVinci 项目，并配置分辨率、帧率和码率。

关键体验要求：

- 自检清单显示通过数量、警告数量和错误数量。
- 错误项提供返回修改入口。
- 高风险项可阻止导出；警告项允许用户忽略并继续。
- 导出任务进入队列，并显示进度、失败原因和重试入口。

## 4. 数据与接口视图

沿用原规格文档中的核心模型：

- `Project`
- `Stage`
- `Script`
- `Shot`
- `Character`
- `Scene`
- `AudioClip`

建议补充前端视图模型：

- `EpisodeProgress`：聚合单集在各阶段的完成度、异常数、锁定状态和最后更新时间。
- `GenerationJob`：统一展示 AI 生成队列、任务状态、失败原因、重试入口和结果版本。

阶段间数据依赖固定为：

```text
拉片报告 -> 剧本 -> 分镜 -> 资产 -> 分镜图 -> 视频/音频 -> 时间线 -> 导出
```

项目级与单集级数据边界：

- 项目级：项目设置、风格词、角色资产、场景资产、道具资产、风格板。
- 单集级：剧本、分镜、分镜图、视频片段、音频片段、字幕、时间线、导出结果。

## 5. 全局交互规范

### 5.1 AI 生成模式

所有 AI 生成功能统一为以下交互结构：

- 自动填充提示词。
- 允许用户编辑提示词。
- 显示模型、数量、尺寸、时长等参数。
- 点击生成后进入生成队列。
- 生成结果以多版本形式展示。
- 用户选择一个版本作为当前使用版本。
- 用户可重新生成、微调提示词、回滚版本。
- 通过审核后锁定。

### 5.2 状态流转

```text
draft -> generating -> generated -> approved -> locked
                         |
                         v
                      rejected -> generating
```

状态含义：

- `draft`：草稿，尚未生成或未填写完成。
- `generating`：生成中。
- `generated`：已生成，待审核。
- `approved`：已通过。
- `rejected`：已拒绝，需要重新生成或修改。
- `locked`：已锁定，作为后续阶段输入。

### 5.3 版本控制

- 每个资产支持多版本。
- 自动保存历史版本。
- 可回滚到任意版本。
- 支持左右分屏对比。
- 明确标记当前使用版本。
- 锁定版本进入后续阶段。

### 5.4 快捷键

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl+G` | AI 生成当前项 |
| `Ctrl+Enter` | 确认当前项，进入下一个 |
| `Ctrl+Z` | 撤销 |
| `Ctrl+Shift+Z` | 重做 |
| `Space` | 播放/暂停 |
| `←` / `→` | 上一个/下一个镜头 |
| `Ctrl+S` | 保存 |
| `Ctrl+E` | 导出 |

## 6. 验收与测试场景

- 新建项目后，确认风格词、集数、画幅、目标平台正确进入工作台，并能在顶部持续展示。
- 逐阶段走通单集流程：上传拉片视频、生成剧本、生成分镜、生成资产、融合图片、生成视频、生成音频、自动排入时间线、完成导出。
- 验证批量生成队列：生成中、失败、重试、取消、重新生成、选择版本、锁定版本状态都正确显示。
- 验证前置阶段未完成时跳转，会出现确认提示。
- 验证已锁定资产被修改时，会提示影响后续镜头和导出结果。
- 验证质量检查和自检清单能阻止或提示高风险导出，但允许用户“忽略并继续”。
- 验证快捷键：播放暂停、保存、生成当前项、确认并进入下一项、上下镜头、撤销重做、导出。

## 7. 当前假设

- 这是一个桌面优先的创作者生产工具，不做营销式首页。
- 项目级资产跨集复用，单集内容独立推进。
- AI 生成结果必须经过人工审核才能锁定并流入下一阶段。
- 本文档只描述前端 UI/UX 与实现结构，不包含后端 API、模型调用和真实素材生成细节。
