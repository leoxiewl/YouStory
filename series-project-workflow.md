# 个人人生故事剧集生产功能计划

## 1. 设计原则

本产品的下一阶段不再只把用户的一段回忆生成成单个短片，而是把“个人人生故事”组织成可持续生产的剧集。

核心原则如下：

1. **项目即剧集**：一个“项目”就是一部完整的个人故事剧集，而不是单条视频。
2. **剧集可拆分为多集**：一个项目下可以包含第 1 集、第 2 集、第 3 集，直到最终集。
3. **每集都有完整创作流程**：每一集都独立经历剧本、分镜、视频、音频、剪辑五个环节。
4. **人工智能负责提效，人负责定稿**：人工智能输出初稿、候选方案和自动化资产，用户保留选择、调整和确认权。
5. **先固定主流程，再扩展子能力**：第一版先稳定五段生产链路，后续再增加版权审核、角色库、资产库、发布、协作等模块。

## 2. 层级结构

```
项目（完整剧集）
├── 第 1 集
│   ├── 环节 1：剧本创作
│   ├── 环节 2：分镜设计
│   ├── 环节 3：视频生成
│   ├── 环节 4：配音及音效
│   └── 环节 5：后期剪辑
├── 第 2 集
│   ├── 环节 1：剧本创作
│   ├── 环节 2：分镜设计
│   ├── 环节 3：视频生成
│   ├── 环节 4：配音及音效
│   └── 环节 5：后期剪辑
└── 最终集
    └── 同样拥有完整创作流程
```

### 项目

项目是最高层级。一个项目代表一部完整的“个人人生故事”剧集。

项目需要承载：

- 剧集标题
- 故事主题
- 人生阶段或叙事范围
- 目标集数
- 整体风格
- 主要人物
- 关键人生节点
- 全局创作状态

### 剧集单集

每一集是项目下的独立生产单元。

单集需要承载：

- 集数编号
- 单集标题
- 单集摘要
- 本集关键词
- 本集情绪基调
- 本集主要冲突或转折
- 本集工作流状态
- 本集生成资产

### 创作流程

每一集都拥有一套完整工作流。项目可以汇总所有单集进度，但每个阶段的真实状态属于具体单集。

建议的阶段状态：

- 未开始
- 生成或编辑中
- 待用户确认
- 已确认
- 需要修改

## 3. 单集完整创作流程

### 第一步：剧本创作环节

目标：把关键词、人生记录、网络文学版权素材或长文本源材料快速转成可拍摄、可分镜的剧本初稿。

人工智能能力：

- 根据关键词快速生成单集初稿。
- 将网络文学版权素材或长文本素材短时间转化为脚本。
- 保留原素材中的“爽点”、情绪高点、反转点和人物动机。
- 提供多个剧情分支，供用户选择不同走向。
- 将数周级别的剧本拆解与改写工作压缩到数小时级别。

用户可调整内容：

- 单集主题
- 关键词
- 人物关系
- 叙事视角
- 高光片段
- 情绪强度
- 结尾走向

主要输出：

- 单集剧本初稿
- 剧情分支候选
- 人物与场景列表
- 高光与爽点清单
- 可供分镜使用的场景段落

### 第二步：分镜设计环节

目标：把剧本转化为可视化镜头设计，降低前期美术和分镜成本。

人工智能能力：

- 输入剧本后自动输出高分辨率画面候选。
- 支持超高清分辨率目标的画面规划。
- 支持武侠、国漫、电影感、写实、插画等风格化渲染。
- 用户可以实时调整角色、构图、服装、场景、光线和镜头细节。
- 将文字剧本拆解成镜头、画面和视觉连续性要求。

用户可调整内容：

- 画面风格
- 镜头景别
- 角色造型
- 场景元素
- 色彩氛围
- 镜头顺序

主要输出：

- 分镜帧
- 镜头列表
- 每个镜头的画面提示词
- 角色一致性参考
- 场景与美术参考

### 第三步：视频生成环节

目标：把剧本和分镜资产转化为可剪辑的视频片段。

人工智能能力：

- 使用文字生成视频模型，从剧本或镜头描述生成视频。
- 使用图片生成视频模型，从分镜帧生成动态镜头。
- 支持部分平台的“首尾帧生成视频”能力。
- 通过首帧和尾帧约束，缓解角色动作不稳定、身份漂移和镜头连续性问题。
- 为每个镜头生成多个候选片段。

用户可调整内容：

- 动作幅度
- 镜头运动
- 人物表演
- 视频时长
- 首尾帧
- 候选版本选择

主要输出：

- 单镜头视频片段
- 片段候选版本
- 首尾帧约束记录
- 角色与场景一致性问题标记
- 进入剪辑阶段的视频素材包

### 第四步：配音及音效环节

目标：为单集生成贴合人物、情绪和剧情节点的声音层。

人工智能能力：

- 支持多音色配音和情感演绎。
- 根据角色、年龄、性格和情绪生成不同声线。
- 自动匹配背景音乐。
- 根据剧情节点添加环境声、动作音效和情绪音效。
- 一键切换方言或外语版本，支持内容出海。

用户可调整内容：

- 旁白音色
- 角色声线
- 情绪强度
- 背景音乐风格
- 音效密度
- 方言或外语版本

主要输出：

- 旁白音轨
- 角色对白音轨
- 背景音乐方案
- 剧情节点音效
- 多语言或方言音频版本

### 第五步：后期剪辑环节

目标：把视频、配音、背景音乐和音效整合为可预览、可导出的单集成片。

人工智能能力：

- 自动拆镜和整理镜头素材。
- 根据节奏和剧情节点完成初剪。
- 自动卡点音效和背景音乐。
- 修复画面闪烁、变形、扭曲等生成问题。
- 输出多个剪辑节奏版本，让用户选择。

用户可调整内容：

- 镜头顺序
- 剪辑节奏
- 字幕样式
- 背景音乐起止点
- 音效卡点
- 画面修复优先级

主要输出：

- 单集初剪版本
- 修复任务清单
- 字幕和音频对齐结果
- 可预览成片
- 最终导出版本

## 4. 推荐数据模型

```
故事项目：
  编号
  标题
  摘要
  剧集主题
  目标集数
  单集列表
  状态：草稿、制作中、已完成
  创建时间

故事单集：
  编号
  所属项目编号
  集数
  标题
  摘要
  关键词列表
  情绪基调
  创作流程列表

单集创作流程环节：
  编号
  环节名称：剧本创作、分镜设计、视频生成、配音及音效、后期剪辑
  状态：未开始、生成或编辑中、待用户确认、已确认、需要修改
  输入材料
  输出材料
  备注
```

## 5. 第一版实现边界

第一版可以先实现结构和流程管理，不必立刻接入真实人工智能生成。

建议优先级：

1. 在产品中把项目展示为完整剧集。
2. 允许项目包含多个单集。
3. 每集展示五段创作流程。
4. 支持每个流程阶段维护状态、输入、输出和备注。
5. 为剧本创作阶段预留关键词、初稿、剧情分支和爽点清单字段。

暂不做：

- 真实超高清图片生成
- 真实视频模型接入
- 真实文字转语音或背景音乐自动匹配
- 自动剪辑和画面修复
- 版权审核和发布分发

## 6. 后续可扩展方向

- 角色库：维护人物外貌、声音、关系和成长变化。
- 场景库：维护家庭、学校、城市、旅行地点等长期复用场景。
- 风格库：维护国漫、写实、纪录片、电影感等视觉风格。
- 资产库：统一管理剧本、分镜、视频、音频、字幕和成片。
- 剧情树：把多个剧情分支保存为可回溯版本。
- 出海版本：为不同语言、地区文化和字幕规范生成本地化版本。
